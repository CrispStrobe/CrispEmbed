# CrispEmbed regression suite

Pinned, reproducible parity + smoke checks for every engine we can confirm.
Mirrors CrispASR's `tests/regression/` but adapted to CrispEmbed's per-engine
`test-<engine>-diff` harnesses and the `crispembed` CLI.

## Files

- **`manifest.json`** — the source of truth. One entry per engine, pinning the
  GGUF under test (repo + revision SHA + filename) so an upstream re-quantise
  can't silently regress us, plus per-tier check config.
- **`run_one.py`** — downloads the pinned artifacts and runs one entry (or a
  whole tier). Also a library: `parse_diff_stdout`, `evaluate_diff`,
  `run_diff`, `run_smoke`, `hf_download` are imported by the Kaggle kernel.

## Tiers

| tier | what runs | pass condition |
|---|---|---|
| `diff` | `test-<engine>-diff <model> <ref>` | every parsed stage's `cos_min ≥ threshold` (`diff_thresholds`, `"*"` = global default, per-stage overrides; `require_stages` must be present) |
| `smoke` | the `crispembed` CLI on a fixed input | output is sane — `embedding` of the right `dim`, finite, non-zero norm (or a `golden`/`nonempty` match) |

`diff` is the gold standard and needs a frozen `*-ref.gguf` on HF. `smoke` is
the **confirmation tier** for engines that don't have a ref yet — it proves the
engine loads and emits a correctly-shaped result. Promote an engine from `smoke`
to `diff` once a ref is baked. See `docs/diff-harness-coverage.md` (generated by
`tools/audit_diff_coverage.py`) for the live gap list.

## Reference-archive convention

Refs live in **each model's own HF GGUF repo**, not a central fixtures repo
(unlike CrispASR). New refs go under `diff-harness-ref/<engine>-ref.gguf`
(`cstr/qari-ocr-crispembed-GGUF` already follows this). Legacy refs at the repo
root (`instructir-ref.gguf`, `hat-ref.gguf`, `pan-ref.gguf`,
`granite-vision-ref.gguf` + `granite-llm-ref.gguf`) are pinned as-is.

## Running locally

```bash
# 1. Build the CLI + the diff binaries you want to check
cmake --build build --target crispembed test-pan-diff test-instructir-diff \
    test-hat-diff test-granite-vision-diff -j

# 2. Validate manifest pins without downloading anything
python tests/regression/run_one.py --dry-run

# 3. Run one entry (needs huggingface_hub + a token in ~/.cache/huggingface/token
#    or $HF_TOKEN; point HF_HOME at a writable cache)
HF_HOME=/tmp/ce-hf python tests/regression/run_one.py \
    --backend pan --build-dir build

# 4. Run a whole tier
python tests/regression/run_one.py --tier diff  --build-dir build
python tests/regression/run_one.py --tier smoke --build-dir build
```

## Running everything (Kaggle)

The heavy backends (granite-vision q8_0 ~2.7 GB, the VLM OCR models) and the
full sweep are too big for local iteration. `tools/kaggle/crispembed-regression/`
runs the whole manifest on Kaggle in `validate` mode, and re-bakes refs in
`rebake` mode. It pushes the rolling progress **and each completed engine's
result** to the `cstr/crispembed-kaggle-progress` HF dataset after every step,
so a crashed/timed-out kernel still leaves every finished verdict on HF.

## Adding an entry

1. Pick the engine's HF GGUF repo + a pinned revision SHA + the GGUF filename.
2. **`diff` tier**: confirm a `test-<engine>-diff` target exists (`CMakeLists.txt`)
   and a `*-ref.gguf` is uploaded; add `ref`, `argv`, `diff_thresholds`.
   If no ref yet, bake one with `tools/dump_<engine>_reference*.py`
   (a `*_from_gguf` dumper needs no source-model download) and upload it.
3. **`smoke` tier**: add a `smoke` block with the CLI `args` (use `{model}` for
   the GGUF path) and an `expect` spec.
4. `python tools/audit_diff_coverage.py --probe-hf` to refresh the coverage doc.
